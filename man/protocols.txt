PROTOCOLS(5)                   File Formats Manual                  PROTOCOLS(5)

NAME
     protocols – protocol name data base

DESCRIPTION
The protocols file contains information regarding the known protocols used in the DARPA Internet.
プロトコルファイルには、DARPAインターネットで使用されている既知のプロトコルに関する情報が含まれています。
For each protocol a single line should be present with the following information:
各プロトコルについて、以下の情報に1行が存在する必要があります。

 official protocol name
 protocol number
 aliases

Items are separated by any number of blanks and/or tab characters.
項目は、任意の数の空白文字やタブ文字で区切ります。
A ``#'' indicates the beginning of a comment; characters up to the end of the line are not interpreted by routines which search the file.
``＃ ''はコメントの始まりを示します。行の終わりまでの文字は、ファイルを検索するルーチンによって解釈されません。

Protocol names may contain any printable character other than a field delimiter, newline, or comment character.
プロトコル名には、フィールド区切り文字、改行、またはコメントの文字以外の印刷可能文字が含まれている可能性があります。

INTERACTION WITH DIRECTORY SERVICES
Processes generally find protocol records using one of the getprotoent(3)
family of functions.
On Mac OS X, these functions interact with the DirectoryService(8) daemon, which reads the /etc/protocols file as well as
searching other directory information services to determine protocol name
and number information.

FILES
     /etc/protocols

SEE ALSO
     getprotoent(3), DirectoryService(8)

HISTORY
     The protocols file format appeared in 4.2BSD.
